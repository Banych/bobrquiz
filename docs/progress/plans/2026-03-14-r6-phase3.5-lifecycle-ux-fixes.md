# R6 Phase 3.5 — Game Lifecycle & UX Fixes

**Date:** 2026-03-14
**Status:** ✅ Complete
**Depends on:** R6 Phase 3 ✅

---

## Overview

A set of five bugs and UX gaps discovered during manual testing after Phase 3 shipped. These all touch the game lifecycle (quiz status transitions), host controls, player answer UI, and presence/connection tracking. None require new infrastructure — all are fixable within existing layers.

**Issues found:**
1. Completed quiz has no way to restart or reset to a startable state — only delete is available
2. "Resume Quiz" button on host dashboard always shows for started quizzes, but throws on press; should be conditional or replaced with contextual actions (Finish / Restart)
3. Player answer UI doesn't adapt to question type — options should show when available, free-text when not
4. No "Start Hosting" shortcut from the admin quiz detail page — host must navigate to `/host` first
5. Connected players transition to `away → disconnected` even when actively answering; status never reverts to `connected`

---

## Goals

- [x] Quiz lifecycle: allow resetting a completed quiz back to `Pending` for replay
- [x] Host dashboard: replace always-visible "Resume" with contextual action buttons based on quiz status
- [x] Player page: render answer options that match the current question type (multiple choice vs free-text)
- [x] Admin quiz page: add "Open Dashboard" / "Open Live View" buttons
- [x] Presence: fix away → disconnected regression so active players stay `connected`

**Solution:** Add a `ResetQuizUseCase` that transitions a quiz from `Completed` (or `Active`) → `Pending`, clears `startedAt`/`endedAt`/`currentQuestionIndex`, and clears player answer state.

## Technical Decisions

**Step 1 — Reset quiz:**
Reset to `Pending` (the pre-start state) so the quiz can be started again. `Quiz.reset()` also clears `startedAt`, `endedAt`, `currentQuestionIndex`, unlocks all questions, and wipes accumulated answers — giving a clean slate for a replay.

**Step 2 — Button logic:**
Keep changes in the presentation layer only. No new domain methods needed for finish/status display.

**Step 3 — Question type rendering:**
The `currentQuestion` DTO should already carry `type` and `options`. If options are missing from the DTO, extend `CurrentQuestionDTO` to include them — this is an application-layer DTO change, not a domain change.

**Step 5 — Presence fix:**
Prefer fixing the heartbeat/away threshold over patching the display layer. A display-only fix would mask the underlying issue.
